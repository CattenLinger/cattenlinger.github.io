用 tinc 搭建可桥接的私人内网

Posted on 2020-10-28 In 应用实践 , 网络搭建

​用 tinc 有好一段时间了。我的服务大多数都使用 tinc 组成的内网来承载。一直以来我是直接固定 IP 使用，设置成「单地址」模式。因为网络内不止固定 IP 的服务器，对于临时进入内网的设备来说这种方式就十分麻烦了：每新增一个设别，需要翻查哪些 IP 可用，然后分别设置服务器端和客户端的「客户端配置文件」。

​tinc 除了路由模式单地址使用，还能使用其他模式，可在设置文档内查阅其模式类型：

​switch 和 hub 都比较类似，一般建议使用 switch。switch 模式下 tinc 相当于一个交换机，可以有交换机功能；而 hub 模式仅仅只是一个 hub ，就像一个对水流什么都不做的管道，同时把所有包都广播到所有节点让各个节点自己看着办：对各个节点而言，内网流量一览无遗，但同时增加很多流量消耗，对一般组网使用来说没什么必要。使用交换机模式，可以传递二、三层流量，可实现 dhcp 、广播等功能，相当于一个普通路由器，对比起来对小型私人内网来说，实用性比较高。

接下来开始介绍一下如何使用 switch 模式搭配 dhcpd 来搭建一个 tinc 内网。

网络结构

​一台服务器，多个客户端，而且客户端还能再靠自己的 tinc daemon 配合桥再拖一堆客户端。如上是 daemon 直接的结构，网络拓扑上大家都是平级的，在同一个内网的。

配置桥来共享内网

在客户端，通过配置桥来共享 tinc vpn 内网是个好玩同时也挺实用的做法，例如：电脑上有虚拟机，这样可以把虚拟机接进桥然后与在其他地区的节点通讯；有多余的网口，与无线 AP 或者交换机连接，连接上 AP 或者交换机的设备能直接获得 IP 并访问内网。

不同的系统有不同的配置方式，这里我以 ubuntu 18.04 为例，假设客户端电脑上有个多余的网口 eth1 用来接 AP。
